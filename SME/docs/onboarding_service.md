# Onboarding Service (SME)

## Purpose
Manage customer registration, profile persistence, and initial eligibility classification.

## Base URL
`http://localhost:8001`

## Endpoints
- `GET /health`: service liveness
- `POST /customers`: create customer profile
- `GET /customers`: list customers
- `GET /customers/{customer_id}`: fetch single customer

## Business Rules
- Customer reference generated by phone and credit-score seed.
- Eligibility defaulting:
  - `ELIGIBLE` if credit score >= 700
  - `REVIEW` otherwise

## Dependencies
- PostgreSQL schema: `onboarding`
- Shared model contracts from `backend/shared/models/entities.py`

## Failure Modes
- Duplicate email/phone can violate uniqueness constraints.
- Invalid payloads rejected by Pydantic validation.
